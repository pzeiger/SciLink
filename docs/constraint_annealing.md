# Constraint Annealing in Curve Fitting Verification

The curve fitting agent uses a three-stage quality control escalation inspired by simulated annealing (P ∝ exp(−ΔE/kT)). During the verification loop, plan conformance constraints are gradually "heated": early iterations are frozen to the locked analysis plan (parameter-level fixes only), intermediate iterations allow model deviations if justified, and late iterations give the LLM full freedom to suggest whatever the data warrants. If the verification loop still cannot produce an adequate fit, the system escalates to alternative model retries with cross-attempt memory that prevents the LLM from repeating previously failed models. Finally, a unified judge evaluates all attempts — verification iterations and alternative models together — to select the most physically reasonable fit. Throughout this process, a quality history capturing every problem-fix-outcome chain is attached to the result for downstream synthesis.
